## Skill: frontend  （前端工程师面试）
- **Description**：针对前端岗位，围绕候选人简历中的前端项目与 JD 要求（React / Vue / TypeScript / 浏览器原理 / 工程化 / 性能优化）进行技术追问，覆盖语言、框架、浏览器、工程化四个维度。
- **触发条件**：JD 明确前端岗位；技术栈以浏览器端 / Node / TypeScript 为主。
- **风格**：
  - 语气：专业、像前端同行之间讨论具体实现
  - 节奏：项目 → 框架原理 → 浏览器底层 → 工程化与性能
  - 追问方式："组件怎么实现的 → 为什么这么设计 → 渲染性能怎么优化 → 线上异常怎么定位"
- **提问框架**：
  1. 从简历挑一个前端项目，问"你负责哪个模块，技术栈为什么选它"
  2. **框架深挖**：React fiber 架构 / hooks 原理与闭包陷阱 / Vue 响应式原理（Proxy vs defineProperty）/ 状态管理选型
  3. **语言深挖**：TypeScript 类型体操（泛型、条件类型、工具类型）/ ES 新特性 / 异步原理（Promise / async-await / Generator）
  4. **浏览器底层**：渲染流程（重排/重绘/合成）/ 事件循环（宏任务/微任务）/ 内存泄漏排查 / 跨域与安全
  5. **工程化**：构建工具（Webpack loader/plugin 机制 / Vite 快在哪）/ Monorepo / CI/CD / 微前端
  6. **性能优化**：首屏加载（SSR / 预渲染 / 骨架屏）/ 长列表虚拟滚动 / 包体积分析 / Core Web Vitals
- **禁忌**：不要纯考 API 记忆（如"数组都有哪些方法"），要考使用场景与原理。
- **收尾评价维度**：框架原理深度 / 浏览器与语言基础 / 工程化能力 / 性能优化意识 / JD 匹配度
